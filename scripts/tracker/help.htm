Before you start installing there are a few things you should know:

  • This script requires a MySql backend.
  • This script relies on you to import the tables and their data into your database.
  • The scripts need to know your MySql data to connect.

The first step for using this script is to create the needed tables in your MySql database, for this i have included a file called scounter.sql this file contains the SQL statements to create the needed tables. If you are using phpMyAdmin or a simulair product to maintain your database then you can choose to select the file scounter.sql for execution. The table will then automaticly be created.

When the table is created you need to configure 2 php scripts to match your needs.

The first to edit is sc_addhit.php, alter the following settings to reflect your database configuration:

$SQL_HOST="localhost";
$SQL_USER="MyUserName";
$SQL_PWD="MyPassWord";
$SQL_DB="MyDatabase";

These four settings are the most important settings needed for contacting your database, usualy LocalHost doesnt need change. However the username, password and database name are required. The following four settings reflect the names of the tables needed. If you simply imported the SQL file mentioned above then these settings dont need changing. (if you want to keep stats from different sites in the same database you could edit the sql file and rename the counter and log table, in that case also change the table names below.)


$SQL_COUNTTABLE="SC_COUNTER";
$SQL_LOGTABLE="SC_LOG";
$SQL_REFERENCETABLE="SC_REFERENCE";
$SQL_BLOCKIPTABLE="SC_BLOCKIPS";

Also you can enable / disable the icon displayed when logging hits on a page, by default the icon is enabled, set the value to false if you wish to disable it.

$showicon = true;

The file sc_information.php contains the following settings that require configuration:

$SQL_HOST="localhost";
$SQL_USER="MyUserName";
$SQL_PWD="MyPassWord";
$SQL_DB="MyDatabase";

These settins are the same as the settings for the addhit.php script. Again you can change the table names but this is only needed if you changed them in the sql script.


$SQL_COUNTTABLE="SC_COUNTER";
$SQL_LOGTABLE="SC_LOG";

These are the color settings for the statistics page. They offcourse are optional.

$TABLE_HEAD_COLOR = "#CCCCCC";
$TABLE_COLOR = "#0099CC";
$TEXT_COLOR = "#000066";

Thats it! If you imported the sql script and edited the php scripts as mentioned then all configuration is done and everything should be ready for usage!
 
If you are using a static IP address then you can block it from the counter by adding the ip address to the SC_BLOCKIPS table. All IP's entered in that table will be ignored by the counter.

The Super Counter can count for multiple pages at once, it updates the 'global' counter and also the specific page counter. If you want a page to be counted when hit then enter the following line of code into your HTML file:

<?php include "sc_addhit.php"; ?>

This makes the script log the hit and displays the counter icon. (note that the ALT text in the icon gives you information on the counters action taken.

If you wish to display information from the counter on your page then you always need to include the following tag:

<?php include "sc_information.php"; ?>

Once thats done the functions inside the script are available for use, the following information require the following tag:

Information Tag
Total hits on this site <?PHP show_totalhits(); ?>
Total hits to this page <?PHP show_totalhits_currentpage(); ?>
Total hits today <?PHP show_todayhits(); ?>

Total hits this month

 <?PHP show_thismonthhits(); ?>
Currently online <?PHP show_currentlyonline();?>

 

More statistics can be found by accessing the page sc_stats.php
